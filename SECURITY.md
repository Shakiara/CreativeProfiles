# Security Policy

## Supported Versions

Security fixes are prioritized for the latest public version of Creative Profiles.

| Version | Supported |
| --- | --- |
| 1.1.x | Yes |
| 1.0.x | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issue threads.

Report security concerns privately to the project maintainer. Include as much detail as possible:

- Creative Profiles version
- Minecraft and NeoForge versions
- Server environment
- Steps to reproduce
- Impact of the vulnerability
- Relevant logs or crash reports

## What Counts as Security-Relevant

Examples include:

- Ways for players to duplicate Survival items through profile switching.
- Ways for players to access another player's saved profile data.
- Permission bypasses for admin-only commands.
- Server crashes triggered by normal player actions.
- Unsafe behavior involving persistent player data.

## Response Expectations

Security reports will be reviewed as soon as possible. If the issue is confirmed, a fix will be prepared for a future release and credited when appropriate.
